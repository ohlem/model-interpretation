## Разработка модуля предобработки данных с оценкой влияния на качество прогноза и интерпретируемость моделей

Датасет: [Online Retail II (UCI)](https://www.kaggle.com/datasets/mashlyn/online-retail-ii-uci)

Целевая переменная: факт совершения повторной покупки (`Target = 1`) в течение 3 месяцев после даты среза.

Данный проект представляет собой программный модуль для исследования не только качества моделей машинного обучения, но прежде всего роли предобработки данных в итоговом результате. Основной акцент сделан на том, как разные уровни очистки транзакционных данных и разные схемы генерации признаков меняют:

- качество прогноза повторной покупки;
- состав и значимость признаков;
- устойчивость и прозрачность объяснений моделей;
- интерпретацию поведения клиентов.


## Цель и исследовательский фокус

**Цель работы**: исследовать влияние глубины предобработки данных и инженерии признаков на качество прогноза и интерпретируемость моделей машинного обучения в задаче предсказания повторной покупки.

**Исследовательская идея проекта**:

- одна и та же бизнес-задача решается на нескольких версиях одного и того же датасета;
- каждая версия отличается уровнем очистки и набором построенных признаков;
- затем сравнивается не только качество моделей, но и то, как меняются объяснения этих моделей.

Таким образом, проект отвечает на вопрос: **насколько именно предобработка данных влияет на результат моделирования и на доверие к полученным выводам**.

## Основные задачи проекта

- Реализовать несколько уровней пайплайнов предобработки: от минимальной очистки до расширенной.
- Построить несколько наборов клиентских признаков: базовые RFM, поведенческие, временные и страновые.
- Сравнить модели на наборах данных, полученных после разных сценариев предобработки.
- Оценить влияние предобработки не только на метрики качества, но и на интерпретируемость моделей.
- Выявить, какие шаги очистки и какие группы признаков дают наибольший вклад в итоговый результат.

## Почему в проекте важна именно предобработка

В транзакционных данных Online Retail присутствуют следующие проблемы:

- пропуски в `Customer ID`;
- отмененные заказы;
- нулевые и отрицательные значения `Quantity` и `Price`;
- служебные и нетоварные `StockCode`;
- выбросы по цене и количеству.

Если эти особенности игнорировать, модель обучается на шуме, а интерпретация начинает отражать артефакты данных, а не реальные паттерны поведения клиентов. Поэтому в проекте предобработка рассматривается как управляемый экспериментальный фактор, влияющий на:

- структуру обучающей выборки;
- распределение признаков;
- стабильность коэффициентов и важностей;
- вид SHAP-, LIME- и Feature Importance-объяснений.

## Что сравнивается в эксперименте

Эксперимент в проекте построен вокруг **сравнения сценариев предобработки**, а модели выступают как инструмент измерения эффекта этих изменений. Иными словами, здесь сравниваются не просто разные алгоритмы, а разные версии одного и того же датасета после разных уровней очистки и обогащения признаками.

Основной объект анализа:

- как меняется состав данных после каждого сценария очистки;
- как меняется качество прогноза на этих версиях данных;
- как меняются важности признаков и объяснения моделей;
- дает ли более глубокая предобработка устойчивый выигрыш по сравнению с минимальной очисткой.

Базовое сравнение строится на трех уровнях предобработки:

| Пайплайн | Содержание | Исследовательский смысл |
|---|---|---|
| `baseline` | Только удаление строк без `Customer ID` | Контрольный сценарий с минимальным вмешательством |
| `standard` | Удаление отмен, отрицательных `Quantity`/`Price`, служебных `StockCode` | Базовая практическая очистка данных |
| `advanced` | Дополнительно удаление выбросов по IQR для `Price` и `Quantity` | Углубленная очистка и стабилизация данных |

После очистки для каждого пайплайна строится собственный набор признаков. Это важно, потому что в исследовании оценивается не только эффект фильтрации строк, но и эффект последующей инженерии признаков, которая напрямую зависит от качества исходных данных.

Сравниваются три набора признаков:

| Набор признаков | Состав |
|---|---|
| `baseline` | RFM-признаки |
| `standard` | RFM + поведенческие |
| `advanced` | RFM + поведенческие + временные + страна |

Далее на полученных версиях данных обучаются модели:

- Logistic Regression
- Decision Tree
- XGBoost

## Методологический акцент

В текущей реализации проект специально построен так, чтобы уменьшить риск утечки данных:

- разбиение на train/test выполняется по клиентам до fit-зависимых шагов;
- параметры очистки, зависящие от данных, вычисляются только на train-части;
- параметры генерации признаков, зависящие от train-распределения, также вычисляются только на train;
- затем те же fitted-артефакты применяются к test без повторного fit.

Это особенно важно для корректной оценки влияния предобработки: иначе различия между пайплайнами могли бы быть частично вызваны утечкой, а не реальным качеством очистки и признаков.

##  Сравнение моделей на advanced-пайплайне

**Цель**: оценить влияние расширенной очистки данных, включая удаление выбросов по IQR, на качество прогноза и интерпретируемость моделей.

В этом эксперименте фиксируется наиболее глубокий сценарий предобработки (`advanced`), после чего сравниваются модели уже не между разными версиями датасета, а внутри одного и того же расширенного пайплайна.

Конфигурации:

- `advanced` + Logistic Regression
- `advanced` + Decision Tree
- `advanced` + XGBoost

Что сравнивается:

- те же метрики качества: ROC-AUC, F1, Precision, Recall;
- устойчивость качества между линейной, деревообразной и бустинговой моделью;
- интерпретируемость результатов: прозрачность коэффициентов, важностей признаков и локальных объяснений.

Что показывает эксперимент:

- помогает ли агрессивная очистка данных повысить качество моделей;
- не приводит ли удаление выбросов к потере полезной информации;
- как меняется баланс между качеством прогноза и объяснимостью модели;
- какая модель лучше использует данные после расширенной очистки.

## Вклад временных и географических признаков

**Цель**: оценить вклад временных признаков в качество прогноза и интерпретацию модели.

В этом эксперименте фиксируется одинаковая расширенная очистка данных, а сравнение строится только на различии в наборе признаков. Такой дизайн позволяет отделить эффект новых признаков от эффекта более агрессивной фильтрации строк.

Сравнение:

- `standard_features`: признаки уровня `standard`
- `advanced_features`: признаки уровня `advanced` = `standard` + temporal + country

Что проверяется:

- влияет ли временная структура поведения клиента на качество прогноза;
- помогает ли включение доминирующей страны клиента;
- как меняется интерпретация модели при расширении признакового пространства.

Что показывает эксперимент:

- полезны ли признаки `tmp_tenure_days`, `tmp_avg_gap_days`, `tmp_orders_last_90d`, `tmp_fav_dow` и country dummies;
- улучшают ли они метрики качества по сравнению с базовым набором `standard`;
- не добавляют ли эти признаки лишний шум и нестабильность в объяснения.

## Результаты эксперимента

В рамках базового сравнения трех пайплайнов предобработки и трех моделей тестируются 9 комбинаций:

```text
3 пайплайна предобработки × 3 модели = 9 экспериментов
```

Ключевой результат эксперимента состоит в том, что **углубление предобработки дает измеримый эффект**. При переходе от `baseline` к `standard`, а затем к `advanced` меняются и обучающая выборка, и признаки, и итоговое качество моделей.

В результате сравнение показывает:

- минимальная очистка позволяет получить рабочий baseline, но сохраняет в данных больше шума;
- базовая практическая очистка уже дает более стабильное представление данных;
- расширенная очистка с обработкой выбросов и более полным набором признаков чаще дает лучшие значения ROC-AUC;
- эффект предобработки проявляется не только в метриках, но и в том, какие признаки начинают доминировать в интерпретации моделей.

Пример итоговых результатов:

| Pipeline | Model | Features | ROC-AUC | F1 | Precision | Recall | 
|---|---|---:|---:|---:|---:|---:|
| Advanced | Logistic Regression | 19 | 0.8232 | 0.6621 | 0.6211 | 0.7088 |
| Advanced | Decision Tree | 19 | 0.8192 | 0.6573 | 0.7023 | 0.6176 |
| Advanced | XGBoost | 19 | 0.8184 | 0.6555 | 0.7597 | 0.5765 |
| Standard | Logistic Regression | 9 | 0.8124 | 0.6492 | 0.5791 | 0.7386 |
| Baseline | Logistic Regression | 3 | 0.8070 | 0.6683 | 0.6120 | 0.7361 |

## Основные выводы

- **Предобработка действительно влияет на качество модели.**
  Более глубокая очистка и расширение признакового пространства улучшают ROC-AUC по сравнению с минимальной обработкой.

- **Предобработка влияет не только на метрики, но и на объяснимость.**
  После более качественной очистки объяснения становятся менее шумными и лучше отражают реальные поведенческие факторы клиентов.

- **Генерация признаков усиливает эффект очистки.**
  Простого удаления шумных строк недостаточно: заметный прирост дает сочетание очистки и клиентских агрегатов.

- **Интерпретация зависит от версии данных.**
  Один и тот же алгоритм на разных версиях датасета может выделять разные топ-признаки, что делает предобработку важной частью анализа, а не только подготовки данных.

- **Лучшая модель не отменяет важности предобработки.**
  Даже сильные алгоритмы выигрывают от более качественного входного представления данных.

## Структура проекта

```text
retail_preprocessing_module/
├── config.py            # Константы: даты, пути, гиперпараметры
├── data_loader.py       # Загрузка данных и базовый EDA-отчет
├── preprocessing.py     # Fit/apply-пайплайны очистки данных
├── features.py          # Fit/apply-инженерия признаков на уровне клиента
├── models.py            # Обучение моделей и расчет метрик
├── interpretability.py  # SHAP, LIME, Feature Importance
├── experiment.py        # Основной экспериментальный оркестратор
├── report.py            # Визуализация итогов эксперимента
├── requirements.txt
└── data/
```

## Быстрый старт

```bash
# 1. Установить зависимости
pip install -r retail_preprocessing_module/requirements.txt

# 2. Поместить CSV-файл датасета
# online_retail_II.csv -> retail_preprocessing_module/data/

# 3. Запустить все эксперименты
python -m retail_preprocessing_module.experiment

# 4. Запустить только эксперимент 3
python -m retail_preprocessing_module.experiment --experiment exp3

# 5. Запустить эксперимент 4
python -m retail_preprocessing_module.experiment --experiment exp4

# 6. Построить итоговые графики
python -m retail_preprocessing_module.report
```

Все результаты будут сохранены в папке `retail_preprocessing_module/outputs/`.

## Какие артефакты формируются

| `outputs/cleaning_steps.csv` | Подробный отчет о шагах очистки и их влиянии на размер выборки |
| `outputs/interpretability_summary.csv` | Сводка по интерпретации моделей |
| `outputs/plots/metrics_comparison.png` | Сравнение качества моделей |
| `outputs/plots/cleaning_impact.png` | Влияние шагов очистки на данные |
| `outputs/plots/fi_*.png` | Feature Importance для отдельных экспериментов |
| `outputs/plots/shap/*.png` | SHAP-графики |
| `outputs/plots/lime/*.png` | LIME-объяснения |
